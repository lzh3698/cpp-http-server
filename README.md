# C++ HTTP静态文件服务器

一个从零实现的、基于**阻塞I/O+线程池**模型的高并发HTTP/1.1服务器。本项目专注于从底层理解HTTP协议、多线程编程和网络编程核心概念。

## 核心特性
- **并发模型**：采用经典的**多线程阻塞I/O模型**，每个连接由一个独立线程处理。
- **线程管理**：实现**自定义线程池**，避免频繁创建/销毁线程的开销。
- **设计考量**：此模型清晰地展示了HTTP请求-响应的完整过程，是学习网络编程最直观的方式。未来可平滑升级为Reactor模型。

## 编译运行

- **运行**：

- **测试**：

## 技术栈
- C++11 现代C++特性（lambda表达式，标准线程库）
- POSIX Socket API 底层网络编程（系统调用）
- C++多线程
- Linux系统编程（文件I/O，系统调用）

## 实现细节
### 连接处理模型
- **主循环**：主线程持续调用`accept()`接受新连接。
- **连接分发**：每个新连接被封装为任务，提交到线程池的任务队列。
- **阻塞处理**：工作线程从队列获取连接任务，在`read()`/`write()`系统调用上阻塞，直至请求完成。
- **优点**：逻辑清晰，每个连接的上下文独立，便于调试和理解。

### 线程池实现
- **任务队列**：使用`std::queue<std::function<void()>>`，线程安全通过`std::mutex`和`std::condition_variable`保证。
- **优雅关闭**：实现析构函数，等待所有任务完成。

### HTTP协议解析状态机
- **缓冲读取**：使用固定大小缓冲区循环读取socket。
- **状态转移**：实现从“解析请求行”->“解析头部”->“读取发送”的完整状态机。
- **安全性**：对请求路径进行规范化，防止目录穿越攻击（`../`）。

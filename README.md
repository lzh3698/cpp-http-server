                                C++ HTTP静态文件服务器

一个从零实现的、基于 **Reactor模型** 和 **自定义线程池** 的轻量级HTTP/1.1服务器，专注于处理高并发静态文件请求。本项目旨在深入理解网络编程、多线程并发及HTTP协议的本质。

## 🚀 核心特性
- **HTTP/1.1协议解析**：支持GET方法，可解析请求行、头部，正确处理`Content-Length`。
- **高效并发架构**：采用 **主从Reactor模型**（主线程accept + 线程池处理IO）。
- **资源池化**：实现**自定义线程池**（生产者-消费者模型），避免线程频繁创建/销毁开销。
- **安全的静态文件服务**：提供基础路径检查，防止目录遍历攻击。
- **清晰的错误处理**：返回标准的200（OK）、404（Not Found）等HTTP状态码。

二、编译运行

    1、编译：
        make
    2、运行：
        ./http_server
    3、测试
        curl http://localhost:8080/
        （或使用浏览器访问，默认监听8080端口）

三、技术栈

    1、C++11 现代C++特性（lambda表达式，标准线程库）
    2、POSIX Socket API 底层网络编程（系统调用）
    3、C++多线程
    4、Linux系统编程（文件I/O，系统调用）

四、实现细节

    1、线程池实现
        ① 使用工作线程+任务队列模式
        ② 支持任意可调用对象的提交
        ③ 可使用std::future获取异步结果
        ④ 线程安全的队列操作

    2、HTTP协议解析
        ① 支持请求行解析
        ② 正确处理Content-Length
